# A6. Discover cryptographic implementation used offline

## Description
For this activity, I identified examples of cryptography used offline. Offline cryptography refers to cryptographic protection used on local devices, files, storage media, or software without needing an active internet connection.

## Selected examples

### 1. Device login password on a laptop
![Laptop](https://github.com/user-attachments/assets/819afaa2-be2b-472b-8f12-22ac2759c299)

A laptop login password helps protect the device from unauthorised access. Although it looks simple, the system normally stores password-related data securely rather than keeping the password in plain text. This is a basic use of cryptography on a local device.

### 2. File encryption in a ZIP archive
![Password protected Document](https://github.com/user-attachments/assets/bfe58d7d-e91d-449c-8874-9374058a8314)

A password-protected ZIP file is a common offline example of cryptography. The file contents are encrypted, and only someone with the correct password can open them. This helps protect private files when they are stored or shared using a USB drive.

### 3. Encrypted SSD
![SSD](https://github.com/user-attachments/assets/32d04b37-e786-43f5-8293-faa8a4fcc070)

An encrypted SSD is a strong example of offline cryptography. Data stored on the solid-state drive is protected using encryption, which means the files cannot be easily read without the correct password, recovery key, or system authentication. This is useful if the drive is removed, lost, or stolen.


### 4. BitLocker or FileVault
![Encrypted storage](https://github.com/user-attachments/assets/e5eb37a4-a89d-4037-8990-7135c750daae)

BitLocker on Windows and FileVault on macOS are examples of full-disk encryption. They encrypt the drive and protect data at rest. This is an important offline cryptographic implementation because it secures data even when the device is turned off.

### 5. Signed software packages or installers
![software-installer-certificate-info](https://github.com/user-attachments/assets/4c3dfbb2-0aa3-4b2d-a699-057221a1485c)

Some software installers or packages include digital signatures. These signatures help users verify that the software has not been changed and that it came from a trusted publisher. This can still be checked locally after the file has been downloaded.


## Explanation
Cryptography used offline mainly protects data at rest. Unlike online cryptography, which protects data while it is being transmitted, offline cryptography protects files, devices, and local storage. It helps keep information private and prevents unauthorised access or tampering.

## Reflection
This activity showed that cryptography is not only used on websites and online services. It is also built into everyday devices and files. Many people use offline cryptography without noticing it, especially through device encryption and password-protected files.
