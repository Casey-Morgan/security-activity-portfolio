# A18. Discover two hallucination cases when using a generative AI system

## Description
For this activity, I identified two hallucination cases produced by a generative AI system. A hallucination happens when the AI gives information that sounds confident and believable but is false, inaccurate, or unsupported.

## Case 1. Fake reference or source

In the first case, the AI was asked to provide sources for a cybersecurity topic. It returned a list of article titles, author names, and links that looked realistic. However, after checking them, some of the sources did not exist or the details did not match the real publication.

This is a hallucination because the AI created false supporting evidence instead of admitting uncertainty. A user who does not verify the response could incorrectly trust the information.

### Why this is a problem
- it can mislead users into citing false information
- it creates a false sense of accuracy
- it is dangerous in academic, legal, or technical work

### Lesson learned
AI-generated references should always be checked against real websites, databases, or official publications before use.

## Case 2. Incorrect technical explanation

In the second case, the AI was asked to explain a cybersecurity concept or command. It gave a fluent answer, but part of the explanation was incorrect. For example, it may have described a tool option that does not exist, confused two security concepts, or explained a process in the wrong order.

This is a hallucination because the answer sounded technically correct but contained false details.

### Why this is a problem
- it may cause the user to learn the wrong concept
- it may lead to mistakes in practical work
- beginners may not notice the error because the answer sounds confident

### Lesson learned
Technical answers from generative AI should be tested, compared with official documentation, or reviewed using trusted learning materials.

## Evidence
Possible evidence for this activity:
- screenshots of the original AI prompt and answer
- screenshots showing the incorrect part
- screenshot of the correct information from a trusted source
- short comparison notes explaining the difference

## Explanation
These two cases show that generative AI can produce useful responses, but it can also invent facts, references, or technical details. Hallucinations are especially risky when the output looks polished and confident.

## Reflection
This activity showed that generative AI should not be trusted without verification. It can be helpful for brainstorming and drafting, but important claims, references, and technical explanations still need to be checked carefully.
